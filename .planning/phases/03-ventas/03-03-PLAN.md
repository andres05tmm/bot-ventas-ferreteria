```sql
CREATE TABLE IF NOT EXISTS ventas (
    id              SERIAL PRIMARY KEY,
    consecutivo     INTEGER NOT NULL,
    fecha           DATE NOT NULL,
    hora            TIME,
    cliente_id      INTEGER REFERENCES clientes(id),
    cliente_nombre  VARCHAR(300) DEFAULT 'Consumidor Final',
    vendedor        VARCHAR(100),
    metodo_pago     VARCHAR(50),

CREATE TABLE IF NOT EXISTS ventas_detalle (
    id              SERIAL PRIMARY KEY,
    venta_id        INTEGER REFERENCES ventas(id) ON DELETE CASCADE,
    producto_id     INTEGER REFERENCES productos(id),
    producto_nombre VARCHAR(300) NOT NULL,
    cantidad        NUMERIC(10,3) NOT NULL,
    unidad_medida   VARCHAR(50) DEFAULT 'Unidad',
    precio_unitario INTEGER,
    total           INTEGER NOT NULL,
    alias_usado     VARCHAR(200)
);
```

**Header and fail-fast (per D-14):**
```python
"""
migrate_ventas.py — Migra ventas historicas de ventas.xlsx a Postgres.
Ejecutar: railway run python migrate_ventas.py
Idempotente: seguro de re-ejecutar (per D-13).
"""
import os, sys, logging
logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("migrate_ventas")

if not os.getenv("DATABASE_URL"):
    print("ERROR: DATABASE_URL no configurado")
    sys.exit(1)

import db
db.init_db()
if not db.DB_DISPONIBLE:
    print("ERROR: No se pudo conectar a Postgres")
    sys.exit(1)
```

**Read Excel (per D-12 — only the active ventas.xlsx, all sheets):**
```python
import openpyxl
import config

EXCEL_FILE = config.EXCEL_FILE  # "ventas.xlsx"
wb = openpyxl.load_workbook(EXCEL_FILE, read_only=True, data_only=True)
```

**Iterate all monthly sheets:**
Loop through ALL sheets in `wb.sheetnames`. Skip sheets named "Compras", "Registro de Ventas-Acumulado", or any sheet that doesn't look like a monthly sheet (heuristic: has data rows with a fecha column).

For each sheet:
1. Detect column positions using the same header-row logic as `_leer_excel_rango` (row `config.EXCEL_FILA_HEADERS`, match column names case-insensitively)
2. Read all data rows starting from `config.EXCEL_FILA_DATOS`
3. Group rows by consecutivo number (same consecutivo = same sale transaction)
4. For each consecutivo group:
   a. **Idempotency check (per D-13):** `existing = db.query_one("SELECT id FROM ventas WHERE consecutivo = %s AND fecha = %s", (consecutivo, fecha))`. If exists, skip this group.
   b. Compute group total: `sum of all line totals in the group`
   c. INSERT into `ventas`:
      ```python
      row = db.execute_returning(
          """INSERT INTO ventas (consecutivo, fecha, hora, cliente_id, cliente_nombre, vendedor, metodo_pago, total)
             VALUES (%s, %s, %s, %s, %s, %s, %s, %s) RETURNING id""",
          (consecutivo_int, fecha_str, hora_str, cliente_id_pg, cliente_nombre, vendedor, metodo, group_total)
      )
      venta_id = row["id"]
      ```
   d. For each line in the group, INSERT into `ventas_detalle`:
      ```python
      db.execute(
          """INSERT INTO ventas_detalle (venta_id, producto_nombre, cantidad, unidad_medida, precio_unitario, total, alias_usado)
             VALUES (%s, %s, %s, %s, %s, %s, %s)""",
          (venta_id, producto, cantidad_float, unidad, precio_unit_int, total_int, alias_or_none)
      )
      ```

**Data type handling:**
- `consecutivo`: `int(float(str(raw_val)))` — Excel may store as float
- `fecha`: if `datetime` instance, use `.strftime("%Y-%m-%d")`; if string, take first 10 chars
- `hora`: if `datetime` instance, use `.strftime("%H:%M:%S")`; if string, take as-is; if None, use `None`
- `cantidad`: `float(str(raw).replace(",", "."))` with fallback to 1.0
- `total`, `precio_unitario`: `int(float(str(raw).replace(",", ".")))` with fallback to 0
- `cliente_id_pg`: None (per Claude's Discretion — don't try to resolve FK during migration; too unreliable)
- `cliente_nombre`: from "cliente" column, default "Consumidor Final"
- `alias_usado`: from "alias" column if present, else None (per Claude's Discretion — map alias column directly)
- `vendedor`, `metodo`: from respective columns, default ""

**Batch size (per Claude's Discretion): process all rows per sheet, no batching needed** — the Excel file is not large enough to require batching.

**Summary output:**

**Error handling:** Individual row parse errors should be logged as WARNING and skipped (don't abort the whole migration for one bad row). Wrap the per-consecutivo-group block in try/except.
