<task type="auto">
  <name>Task 2: Scope /borrar DELETE to current date only in callbacks.py</name>
  <files>handlers/callbacks.py</files>
  <read_first>

  </read_first>
  <action>
In handlers/callbacks.py, inside the `borrar_si` branch (lines 300-315), the two DELETE statements currently delete by consecutivo alone, without date scoping. This means `/borrar 5` could delete sale #5 from any past date.

Change the two DELETE+execute calls (lines 307-314) from:

```python
rows = await asyncio.to_thread(
    _db.execute,
    "DELETE FROM ventas_detalle WHERE venta_id = (SELECT id FROM ventas WHERE consecutivo = %s)",
    [numero],
)
await asyncio.to_thread(
    _db.execute,
    "DELETE FROM ventas WHERE consecutivo = %s",
    [numero],
)
```

To:

```python
rows = await asyncio.to_thread(
    _db.execute,
    "DELETE FROM ventas_detalle WHERE venta_id = (SELECT id FROM ventas WHERE consecutivo = %s AND fecha::date = CURRENT_DATE)",
    [numero],
)
await asyncio.to_thread(
    _db.execute,
    "DELETE FROM ventas WHERE consecutivo = %s AND fecha::date = CURRENT_DATE",
    [numero],
)
```

Both statements get `AND fecha::date = CURRENT_DATE` appended to the WHERE clause. The subquery in the first DELETE also needs the date filter to avoid orphaning detail rows from a different-date sale with the same consecutivo.

Addresses VEN-01: /borrar must only affect today's sales.
