## Key Implementation Details

### Context-Based Filtering Pattern

All tabs follow this pattern:

```javascript
const { selectedVendor } = useVendorFilter()
const vendorParam = selectedVendor ? '&vendor_id=' + selectedVendor : ''
const { data } = useFetch(`/endpoint?param=value${vendorParam}`, [...deps, selectedVendor])
```

This ensures:
- Refetch triggers when vendor selection changes
- Query string includes vendor_id only when a vendor is selected
- Admins viewing all vendors (no selection) get data from all vendors
- Vendors always see only their own data (enforced at backend)

### Admin-Only UI

The vendor selector is wrapped in a conditional check:

```javascript
function HeaderVendorSelector() {
  const { isAdmin } = useAuth()
  if (!isAdmin()) return null
  // render dropdown with vendedores list
}
```

### Role-Based Logic

- **Vendedor role:** No selector visible, all API calls enforce user ID filtering at backend
- **Admin role:** Selector shown, can toggle between "Todos los vendedores" and individual vendors
- **Backend enforcement:** `get_filtro_efectivo()` dependency handles role-based filtering logic
