# Task 260401-24l: Add JWT Authentication to Dashboard

## Summary

Complete JWT authentication system added to FerreBot dashboard using Telegram Login Widget. Users can now securely log in via Telegram, receive JWT tokens valid for 7 days, and access protected dashboard routes. Backend verifies Telegram signatures cryptographically and database user status.

## What Was Built

### Backend (routers/auth.py)

- **POST /auth/telegram** — Receives Telegram Login Widget callback data
  - Verifies Telegram hash using HMAC-SHA256(SHA256(BOT_TOKEN), data_check_string)
  - Validates auth_date is within 24 hours
  - Queries usuarios table for telegram_id with activo=TRUE
  - Returns 403 if user not found (not in database)
  - Issues JWT token with 7-day expiry signed with SECRET_KEY
  - Returns token, nombre, and rol to client

- **GET /auth/me** — JWT verification endpoint
  - Extracts Bearer token from Authorization header
  - Validates JWT signature and expiry
  - Returns usuario_id, nombre, rol on success
  - Returns 401 on invalid/expired tokens

### Frontend (React Router + Components)

- **src/pages/Login.jsx** — Centered login card with Telegram widget
  - Displays "Ferretería Punto Rojo" branding in red (#C8200E)
  - Injects Telegram Login Widget via script
  - Calls window.onTelegramAuth callback on authentication
  - POST to /api/auth/telegram with user data
  - Saves token and user info to localStorage on success
  - Shows error messages for 403 (user not found) and other failures
  - Displays loading spinner during authentication

- **src/hooks/useAuth.js** — Auth state management hook
  - getToken() — retrieves JWT from localStorage
  - getUser() — parses ferrebot_user JSON from localStorage
  - logout() — clears tokens and redirects to /login
  - isAdmin() — checks if user.rol === 'admin'
  - authHeader() — returns { Authorization: `Bearer ${token}` } for API calls

- **src/components/ProtectedRoute.jsx** — Route guard component
  - Redirects to /login if no token in localStorage
  - Calls GET /auth/me to verify token validity
  - Shows loading spinner during verification
  - Renders children if authenticated

- **src/App.jsx** (modified) — React Router integration
  - Wraps app in BrowserRouter
  - Route /login: unprotected Login page
  - Route /*: ProtectedRoute wrapping Dashboard
  - Maintains existing tab-based dashboard functionality

### Infrastructure

- **requirements.txt** — Added PyJWT>=2.8.0 for JWT encoding/decoding
- **dashboard/package.json** — Added react-router-dom ^6.28.0 for client-side routing
- **api.py** — Imported and registered auth router with FastAPI app

## How It Works

### Login Flow

1. User visits http://localhost:5173/login or is redirected from protected route
2. Login.jsx renders centered card with Telegram widget
3. User clicks Telegram button
4. Telegram sends user data (id, first_name, username, photo_url, auth_date, hash) to window.onTelegramAuth
5. Login.jsx POSTs to /api/auth/telegram
6. Backend verifies hash, checks database, issues JWT
7. JWT token and user info stored in localStorage
8. User redirected to / (dashboard)

### Protected Route Access

1. User navigates to dashboard (/), ProtectedRoute mounts
2. ProtectedRoute checks localStorage for token
3. If token exists, calls GET /api/auth/me to verify
4. On success: renders dashboard
5. On failure or no token: redirects to /login

### Token Verification

- Backend validates JWT signature with SECRET_KEY
- Checks exp claim for expiry (7 days from issue)
- Returns 401 on invalid/expired tokens
- Frontend catches 401 and clears localStorage, redirects to /login

  - Example: `openssl rand -hex 32`
- **BOT_TOKEN** — Already exists in Railway environment
- Database — usuarios table must have: usuario_id, telegram_id, nombre, rol, activo columns

## Known Limitations

- Telegram widget is embedded via script injection (standard approach)
- Tokens are stored in localStorage (vulnerable to XSS) — for production consider httpOnly cookies
- No refresh token mechanism (tokens valid for 7 days only)
- No logout endpoint — logout is client-side (token removal)
