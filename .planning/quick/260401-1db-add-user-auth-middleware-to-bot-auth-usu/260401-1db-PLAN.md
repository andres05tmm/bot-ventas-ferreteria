# Quick Plan: Add User Auth Middleware to Bot

## Objective

Implement Phase 2 user authentication gate:
1. Create `auth/usuarios.py` with DB functions to fetch/register/create users
2. Modify `handlers/mensajes.py` to enforce `get_usuario()` gate on all messages
3. Add two command handlers (`/confirmar`, `/registrar_vendedor`) to register telegram IDs
4. Tag all transaction INSERT rows with `usuario_id` (ventas, gastos, compras)
5. Write unit tests covering all auth functions

**Output:** Authenticated message handling, transaction audit trail, 7 passing auth tests.

**Message handler pattern** (`handlers/mensajes.py` ~line 200+ main handler):

- Call `auth.usuarios.get_usuario(telegram_id)` with lazy import
- If None → reply and return (gates the entire message)
- If found → pass `usuario` dict down to INSERT calls
